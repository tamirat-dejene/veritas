Subsystem Directory

Global breakdown of logical subsystems and bounded context limits:

PRESENTATION

Isolates administrative controls and proctor candidate interfaces, decoupling logic routing via standard proxies.

CORES

Core transactional layers (Exams scheduling, Tenants state records, Invoicing logic) are decoupled database-per-service.

Access limits, policies, and tenant validation metrics:

ROUTERS

Gin reverse proxy intercepts tokens and performs active verification against the Session store without calling the DB.

TENANCY

Ensures absolute logical database isolation between exam candidate tenants using structured UUID checks.

Exam lifecycle processing and Machine Learning grading rules:

SESSIONS

Logs candidate answers as key/value matrices. Triggers ready-for-grading event topics on Kafka immediately upon exam submission.

GRADER

Offloads heavy natural language processes asynchronously to specialized evaluator containers, avoiding core CPU locks.

Message queues and asynchronous notification pipelines:

EVENT BUS

Apache Kafka handles eventual database consistency across domains and notifies the Email service to dispatch transactional alerts.

STRIPE WEBHOOKS

Verifies Stripe request payloads to prevent fraudulent subscription state changes across tenant records.
