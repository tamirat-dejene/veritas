Notification Service

Internal Worker

The Notification Service is an event-driven worker responsible for dispatching system-wide communications via Email and other channels. It does not expose public HTTP endpoints and operates primarily by consuming Kafka events from other microservices.

Enterprise and Users Events (Kafka)

Triggered when a new staff member is onboarded. Dispatches a "Welcome Email" containing initial login credentials. Triggers when a user requests a password reset, sending them a secure reset link. Triggered when a new enterprise registration is received. Sends a "Registration Complete" receipt. Triggers when an enterprise is approved. Sends a confirmation email to the contact email address. Triggers when an enterprise is suspended (administrative or payment). Sends a notice to the contact email. Triggers on soft-deletion. Notifies the enterprise about the start of the data retention period. Triggers when an enterprise is permanently deleted after the retention period. Notifies the enterprise contact that their suspended account has been reactivated. Notifies the enterprise contact that their deletion request has been canceled and account restored. Notifies a user that their account has been deactivated by an administrator. Notifies a user that their account has been reactivated. Security notification sent after a user changes their password or resets it via token. Delivers a new temporary password to a user when reset by an enterprise administrator.

Exam Lifecycle Events (Kafka)

Notifies the enterprise administrator when a new exam draft is created. Notifies both administrators and enrolled candidates when an exam window is scheduled. Sends real-time "Go Live" notifications to candidates when an exam becomes available for attempt. Notifies all stakeholders when an examination window officially concludes.

Candidate Events (Kafka)

Triggered when an administrator sends an invitation link to a candidate. Delivers a branded HTML email with a secure, unique invitation link. Triggered when a candidate submits their exam session. Delivers a confirmation email detailing the exam title and submission time. Published when an exam session is ready for evaluation. This is a Fat Event containing the master grading criteria merged with the candidate's answers.

Proctoring Events (Kafka)

Triggered for every behavioral anomaly detected during a session (tab switching, face absence, etc.). Published when a session's cheating probability score is recomputed. Triggers real-time dashboard updates. Published after periodic biometric identity checks. Notifies administrators of identity mismatches.

Tech Stack

  • Language: Go 1.25
  • Communication: Kafka (Sarama)
  • Delivery: SMTP (Gomail)
  • Templating: Go html/template (Embedded)
